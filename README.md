# Water Simulation

To run fully, you need to download visual studio, NVIDIA's CUDA Development Toolkit, and install OpenGL with vcpkg. If you set all those up, this repository should run as-is. Open the WaterSimulation folder in Visual Studio's CMake context, let CMake autogenerate the out/ directory that contains the executable. Then set the run target to local-vcpkg and make sure the executable target is the WaterSimulation.exe generated by CMake and it will run.

If you cannot run it in Visual Studio or otherwise only have a CUDA environment with no OpenGL, you can just take all the .cuh and .cu files in the src directory and run just those as the simulation computations take place entirely within CUDA. However, your only indication that the code is running is the debug print statements as there is no corresponding visualization. Since the simulation is a class it can be instantiated in any file that includes "Simulation.cuh". To run the visualizationless simulation:

```C
#include "Simulation.cuh"

WaterSimulation Simulation(32, 32, 32);

Simulation.InitializeSimulation();

int TimeStep = 1000;

for (int i = 0; i < TimeStep; i++){
    Simulation.Step(true); //For GPU, CPU only is Simulation.StepSerial()
}
```
